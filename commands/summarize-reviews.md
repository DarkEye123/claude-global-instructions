# Process Code Reviews

your task will be to analyze 3 code-review summaries and asses what is really an issue and what is not, then create `final-summary.md` from your findings.
Rate them from 1 to 10 where 10 is most needed fix, 1 lowest. If review propose improvement, asses it same way, but improvements are
always after bug fixes regarding priority. 

Code-review files are `summary-low.md`, `summary-medium.md` and `summary-high.md`. If you can't find them, ask for context the user himself.

Always add original reference you are addressing in the `final-summary.md`. This is export merging all data all together. It needs to be comprehensible, easy to read and process by you but also living human being.
Be thorough and precise. If you find any contradictions, point them out and ask for clarification. You will describe issues in a detailed manner. Do that according to the complexity of the bug or improvement.
If the bug or improvement is simple, you may include just quick notion about it and what to fix or adjust. If the bug or improvement is tricky and needs deep technical understanding, add a great amount of details.
